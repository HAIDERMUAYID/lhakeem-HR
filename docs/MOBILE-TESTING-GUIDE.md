# دليل بدء اختبار النسخة المتجاوبة (الجوال/التابلت)

## 1. تشغيل التطبيق محلياً

```bash
# من مجلد المشروع
cd frontend
npm run dev
```

ثم افتح في المتصفح: **http://localhost:3000** (أو المنفذ الذي يظهر في الطرفية).

> **ملاحظة:** إذا كان الواجهة تتصل بـ API خلفي، تأكد أن الـ backend يعمل وأن عنوان الـ API مضبوط في المتغيرات البيئية (مثلاً `NEXT_PUBLIC_API_URL`).

---

## 2. الاختبار بمحاكاة الجوال (Chrome DevTools)

1. افتح الموقع في **Chrome**.
2. اضغط **F12** (أو Cmd+Option+I على Mac) لفتح أدوات المطور.
3. انقر أيقونة **Toggle device toolbar** (أو Ctrl+Shift+M / Cmd+Shift+M) لتفعيل وضع الجهاز.
4. اختر جهازاً من القائمة، مثلاً:
   - **iPhone 12 Pro** (390×844)
   - **Pixel 5** (393×851)
   - **iPad** (768×1024)
5. حدّث الصفحة إن لزم وجرّب التنقل بين الصفحات.

### ما الذي تتحقق منه؟

- [ ] **تسجيل الدخول:** الحقول وزر "دخول" واضحة وقابلة للضغط بسهولة.
- [ ] **القائمة الجانبية (Drawer):** تفتح من زر القائمة وتغلق بالنقر خارجها أو على رابط.
- [ ] **لوحة التحكم:** البطاقات والرسم البياني يظهران بدون قص أو تداخل.
- [ ] **صفحات القوائم (موظفين، إجازات، غيابات، أجهزة، …):** تظهر كـ **بطاقات (Card View)** وليس جدولاً ضيقاً؛ الأزرار (تعديل، حذف، تفاصيل) سهلة اللمس.
- [ ] **النوافذ المنبثقة (Modals):** تظهر بعرض كامل أو شبه كامل من أسفل؛ زر الإغلاق واضح وكبير.
- [ ] **لا scroll أفقي** في المحتوى الرئيسي (ما عدا الجداول الكبيرة التي لها شريط تمرير واضح).
- [ ] **الهيدر:** زر الإشعارات وزر الخروج يعملان ومساحة اللمس مريحة.

## 3. الاختبار على أجهزة حقيقية (موصى به قبل الاعتماد)

| الجهاز        | المتصفح | الحالة |
|---------------|---------|--------|
| iPhone        | Safari  | ⬜     |
| Android       | Chrome  | ⬜     |
| iPad          | Safari  | ⬜     |
| أندرويد تابلت | Chrome  | ⬜     |

### الوصول من الهاتف إلى النسخة المحلية

1. تأكد أن الهاتف والكمبيوتر على **نفس الشبكة (Wi‑Fi)**.
2. اعرف IP جهازك (مثلاً من إعدادات الـ Wi‑Fi أو تنفيذ `ipconfig` / `ifconfig`).
3. شغّل الواجهة كما في الفقرة 1، ثم من الهاتف افتح:  
   **http://&lt;IP_الكمبيوتر&gt;:3000**  
   مثال: `http://192.168.1.10:3000`
4. إن لم يفتح الموقع، تحقق من جدار الحماية وسماح المنفذ 3000.

## 4. قائمة تحقق سريعة لكل صفحة

عند دخول أي صفحة على عرض جوال (&lt; 768px):

| البند              | نعم / لا |
|--------------------|----------|
| لا قطع نصوص أو تداخل عناصر | ⬜ |
| لا scroll أفقي غير مقصود   | ⬜ |
| الأزرار والروابط الرئيسية ≥ 44px | ⬜ |
| النوافذ المنبثقة تفتح وتغلق وتُغلق بسهولة | ⬜ |
| الإجراءات (إضافة، تعديل، حذف، فلاتر) تعمل كما على الديسكتوب | ⬜ |

## 5. تسجيل النتائج

استخدم **جدول تنفيذ واختبار النسخة المتجاوبة** في الملف:  
`docs/MOBILE-RESPONSIVE-CHECKLIST.md`

حدّث فيه:
- **اختبار الأجهزة:** ضع ✅ بجانب كل جهاز/متصفح بعد الاختبار.
- **اختبار الصلاحيات:** ضع ✅ أو ملاحظات لكل دور (مدير نظام، مدير إجازات، …).
- **الاعتماد النهائي:** بعد انتهاء الاختبار ومعالجة العيوب الحرجة، حدّث القائمة ووقّع الاعتماد.

---

*يمكنك البدء فوراً بتشغيل `npm run dev` من مجلد `frontend` وفتح الموقع في Chrome مع وضع الجهاز (Device Toolbar) لمحاكاة الجوال.*
